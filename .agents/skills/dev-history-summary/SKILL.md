# Dev History Summary

## Overview

Use this skill to summarize recent work from `dev` in three categories:

- Features
- Refactors
- Bugfixes

The summary is generated by `codex exec` from commit messages and bodies, and
defaults to the last 7 days.

## Workflow

1. Run the summary script with the amount of days specified by user, if not specified use 7 days 
2. Use the markdown output as context.
3. Summarize the markdown output to a human product owner and place summaries into three categories (feature, bugfix, refactor) also in markdown
4. Place the summary in chat for the user

## Command

```bash
./.agents/skills/dev-history-summary/scripts/summarize_dev_history.sh
```

## Time Range Options

### Last N days (recommended)

```bash
./.agents/skills/dev-history-summary/scripts/summarize_dev_history.sh --days 14
```

### Force heuristic mode

```bash
./.agents/skills/dev-history-summary/scripts/summarize_dev_history.sh --no-llm
```

### Pick a model

```bash
./.agents/skills/dev-history-summary/scripts/summarize_dev_history.sh \
  --days 14 --model gpt-5
```

### Explicit git-compatible range

```bash
./.agents/skills/dev-history-summary/scripts/summarize_dev_history.sh \
  --since "2026-01-01" --until "2026-01-31"
```

## Branch Selection

Defaults to `dev`, but you can target another branch/reference:

```bash
./.agents/skills/dev-history-summary/scripts/summarize_dev_history.sh \
  --branch origin/dev
```

## Notes

- LLM output is synthesized and may omit low-signal commits.
- If LLM summarization fails, the script falls back to heuristic categorization.
- If `dev` is not present locally, use `--branch origin/dev`.
- Output is markdown, ready to paste in chat.
